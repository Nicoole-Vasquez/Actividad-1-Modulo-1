Elementos de Páginas Web

  • I. Orígenes de la Web
  1. Web 2.0:

    2. Se refiere a la segunda generación del desarrollo y diseño web, caracterizada por la transición de páginas web estáticas (Web 1.0) a contenido dinámico e interactivo, donde los usuarios juegan un papel central en la creación y distribución de contenido. Surgió a principios de los años 2000 y puso un gran énfasis en la conectividad social, la colaboración y la personalización.

  2. Web 3.0 3:

    3. También conocido como la "web semántica" o "web descentralizada," representa la evolución de Internet hacia una red más inteligente, conectada y autónoma. Surgió como una visión para mejorar la manera en que interactuamos con la información y las aplicaciones en línea, con énfasis en tecnologías como blockchain, inteligencia artificial (IA) y datos enlazados.

  3. La W3C y los estándares:

    4. es la principal organización internacional responsable del desarrollo de estándares para la Web. Fundada en 1994 por Tim Berners-Lee, el creador de la World Wide Web, su objetivo es garantizar que la web sea accesible, interoperable y funcional para todos, independientemente del dispositivo o plataforma.

  • II. Lenguajes de etiquetas
  1. SGML:

    2. Es un estándar para definir lenguajes de marcado utilizados para estructurar y representar documentos complejos de manera organizada.

  2. XMLul:

    3. Es un lenguaje de marcado derivado de SGML que permite estructurar, almacenar y transportar datos de manera extensible y legible.

  3. HTML:

    4. Es el lenguaje estándar utilizado para crear y estructurar contenido en la web mediante etiquetas predefinidas.

  4. XHTML:

    5. Es una versión de HTML basada en XML, diseñada para seguir reglas más estrictas de sintaxis y estructuración.

  5. HTML5:

    6. Es la última versión de HTML, que incluye soporte para multimedia, APIs modernas y diseño responsivo, mejorando la funcionalidad de las aplicaciones web.

  • III. Otros lenguajes relacionados al desarrollo web
  1. CSS/CSS3:

    2. Es un lenguaje utilizado para definir el estilo de los elementos en una página web, como colores, fuentes, márgenes y disposición de los contenidos. CSS3 es la versión más reciente de CSS, que incluye nuevas características como transiciones, animaciones, sombras y diseños responsivos.

  2. JavaScript:

    3. Es un lenguaje de programación utilizado para crear interactividad en las páginas web, permitiendo la manipulación dinámica de contenido, eventos y respuestas a las acciones del usuario.

  3. Ajax:

    4. Es una técnica de desarrollo web que permite la actualización asíncrona de una página web sin tener que recargar toda la página, lo que mejora la experiencia del usuario al hacer que la web sea más rápida y fluida.

  4. Tecnologías y lenguajes del lado del servidor:

    5. Son tecnologías y lenguajes utilizados para procesar solicitudes y manejar datos en el servidor antes de enviarlos al navegador del usuario. Algunos ejemplos incluyen PHP, Python, Ruby, Node.js, Java y ASP.NET.

  5. Navegadores:

    6. Son aplicaciones que permiten a los usuarios acceder y visualizar páginas web. Ejemplos populares incluyen Google Chrome, Mozilla Firefox, Safari y Microsoft Edge.

  6. Motores renderizados:

    7. Son los componentes de los navegadores que interpretan el código HTML, CSS y JavaScript de una página web y lo muestran visualmente al usuario. Ejemplos incluyen Blink (Chrome, Edge), Gecko (Firefox) y WebKit (Safari).

  • IV. Herramientas de desarrollo
  1. Software WYSIWYG:

    2. Es un tipo de software que permite a los usuarios crear contenido visualmente sin necesidad de escribir código, mostrando una vista previa en tiempo real de cómo quedará el producto final. Ejemplos comunes incluyen editores de texto y diseñadores web como Adobe Dreamweaver y WordPress.

  2. Otras alternativas:
    • Figma:

      • Es una herramienta de diseño y prototipado colaborativo basada en la web, utilizada principalmente para crear interfaces de usuario (UI) y experiencias de usuario (UX) en aplicaciones web y móviles.

    • InVisión:

      • Es una plataforma de diseño colaborativo que permite crear prototipos interactivos y recibir retroalimentación en tiempo real, siendo ampliamente utilizada en equipos de diseño y desarrollo de productos.

    • Zeplin:

      • Es una herramienta que facilita la colaboración entre diseñadores y desarrolladores, permitiendo exportar diseños desde herramientas de diseño como Figma y Photoshop, y generar especificaciones detalladas para implementar esos diseños en código.

    • GitHub

      • Es una plataforma de desarrollo colaborativo que permite a los desarrolladores almacenar y gestionar código fuente mediante el control de versiones (Git). Es ampliamente utilizada para el desarrollo de software de código abierto y privado.

  3. Inspección de código desde el navegador:

    4. Es una herramienta que permite analizar y modificar el código de una página web directamente desde el navegador. Los desarrolladores pueden inspeccionar el HTML, CSS y JavaScript, hacer cambios en tiempo real y depurar problemas. Herramientas como DevTools de Chrome y Firefox Developer Tools son ejemplos de funcionalidades de inspección de código en los navegadores.

El Diseño De Una Página Web

  • Tipos de páginas Web:
  1. Páginas estáticas: Páginas web con contenido fijo que no cambia a menos que se edite manualmente.
  2. Páginas dinámicas: Páginas cuyo contenido se genera y actualiza automáticamente a partir de datos almacenados en bases de datos.
  3. Páginas de una sola página (SPA): Sitios web que cargan todo su contenido en una única página sin recargar.
  4. Páginas informativas: Sitios web destinados a proporcionar información sobre un tema específico.
  5. Páginas de comercio electrónico (eCommerce): Sitios web que permiten realizar compras en línea.
  6. Redes sociales y foros: Sitios interactivos donde los usuarios pueden crear perfiles y generar contenido.
  7. Páginas de portafolio: Sitios web utilizados para mostrar el trabajo o habilidades de un profesional o artista.
  • Puntos básicos en la elaboración de una página Web:
  1. Definición del propósito: Establecer el objetivo principal de la página web.
  2. Planificación y estructura: Organizar las páginas y contenido del sitio.
  3. Diseño y experiencia de usuario (UX): Crear un diseño atractivo y fácil de usar.
  4. Desarrollo del contenido: Crear contenido relevante y optimizado para SEO.
  5. Selección de tecnología: Elegir las herramientas y lenguajes adecuados para el desarrollo.
  6. Pruebas: Verificar la funcionalidad y compatibilidad del sitio en diferentes dispositivos y navegadores.
  7. Optimización y rendimiento: Mejorar la velocidad y la carga del sitio.
  8. Lanzamiento y mantenimiento: Publicar el sitio y realizar actualizaciones periódicas.
  9. Análisis y mejora continua: Monitorear el rendimiento y ajustar según los resultados obtenidos.

Estructura De Una Página Web

  • Jerarquía de directorio de un proyecto web:

    • La jerarquía de directorio de un proyecto web se refiere a la organización estructural de las carpetas y archivos que componen el proyecto. Un esquema común incluye:

    1. root (raíz del proyecto)
    2. root (raíz del proyecto)
    3. /assets (carpeta para archivos estáticos)
    4. /css (estilos CSS)
    5. /js (archivos JavaScript)
    6. /images (imágenes y gráficos)
    7. /fonts (tipografías)
    8. /pages (páginas adicionales, si las hay)
    9. /lib (librerías externas o frameworks)

    Esta estructura permite mantener organizados los archivos y facilita su acceso, colaboración y mantenimiento.

  • Elementos de estructura para el diseño de una página Web: Concepto

    • Los elementos de estructura de una página web se refieren a la disposición y organización de los componentes en el diseño. Estos incluyen:

    1. Encabezado (Header): Área superior de la página que generalmente contiene el logo, el menú de navegación y otras informaciones clave.
    2. Cuerpo (Body): Contenido principal de la página, donde se incluyen texto, imágenes, videos, etc.
    3. Pie de página (Footer): Sección inferior de la página que usualmente incluye información adicional como derechos de autor, enlaces de contacto o redes sociales.
    4. Navegación (Navigation): Enlaces que permiten a los usuarios moverse entre las diferentes páginas del sitio.
  • Regla utilizada para validar y certificar una página web

    • Para validar una página web, se utiliza el W3C Validator, que verifica que el código HTML, CSS y otros recursos cumplan con los estándares definidos por la World Wide Web Consortium (W3C). La validación asegura que la página sea accesible y compatible con diversos navegadores y dispositivos.
